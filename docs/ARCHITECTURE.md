# 英语能力测评与课程服务系统 - 技术架构文档

## 一、系统概述

### 1.1 项目定位

**目标用户**：3-15岁学生的家长

**核心价值**：从零基础到剑桥考级通关，一站式 AI 英语成长平台

**商业模式**：
```
测评收费(59.9元) → 课程转化 → 考试培训 → 考试报名服务佣金 → 冲刺包/增值服务
```

### 1.2 系统边界

| 终端 | 形式 | 目标用户 |
|------|------|---------|
| 家长/学生端 | 微信小程序 + 手机APP | 3-15岁学生及家长 |
| 访客端 | 扫码 H5 页面 | 潜在用户/线下扫码访客 |
| 学校管理后台 | Web 后台管理系统 | 学校管理员/校长 |
| 教师管理端 | Web + 企业微信集成 | 任课教师/教学顾问 |
| 平台超级后台 | Web 超级管理后台 | 平台运营方 |

### 2.2 技术选型理由

#### 前端技术栈

| 技术 | 选型理由 |
|------|----------|
| **微信小程序** | 微信生态流量入口，用户无需下载，传播便捷 |
| **uni-app** | 一套代码编译到多端(iOS/Android/H5/小程序)，开发效率高 |
| **Vue 3** | 组合式API，响应式，性能优异 |
| **Element Plus** | Vue 3 最佳企业级组件库，定制化能力强 |
| **Vant** | 轻量级移动端组件库，适合H5落地页 |

#### 后端技术栈

| 技术 | 选型理由 |
|------|----------|
| **FastAPI** | Python生态，AI集成最友好，高性能异步框架 |
| **SQLAlchemy** | ORM成熟稳定，支持异步 |
| **MySQL 8.0** | 事务支持强，生态成熟，适合业务数据 |
| **Redis 7.0** | 高性能缓存，会话存储，实时数据 |
| **Elasticsearch** | 题库全文检索，报告搜索 |
| **RabbitMQ** | 异步任务队列，削峰填谷 |

#### AI 能力

| 技术 | 用途 |
|------|------|
| **科大讯飞 ISE** | 口语评测（发音准确度、流利度、完整性） |
| **GPT-4 / 通义千问** | AI外教对话、AI写作评分、报告生成 |
| **IRT 算法** | 自适应测试，精准评估学生水平 |

## 三、核心模块设计

### 3.1 考试板块（核心盈利模块）

#### 模块职责
- 剑桥英语考试信息展示
- 考试报名代理服务
- 全真模拟考试系统
- AI 口语考官
- 通过率保障体系

#### 核心流程

```
考试咨询 → 能力诊断 → 备考建议 → 模拟考试 → 考前冲刺 → 正式报名 → 考试完成 → 成绩发布
```

#### 关键设计

1. **考试状态机**
```
报名待提交 → 已提交审核 → 审核通过 → 报名成功 → 准考证已下载 → 考试完成 → 成绩已出
```

2. **模拟考试计时器**
- 严格按真实考试时间计时
- 听力部分不可回退
- 自动保存答案

3. **AI 口语考官**
- 模拟真实考官行为
- Part 1-4 分层对话
- 实时评分反馈

### 3.2 智能测评系统

#### 模块职责
- 词汇量自适应测试（IRT算法）
- 拼读能力评估
- 听说读写四维评估
- 专业报告生成

#### 核心流程

```
开始测评 → 词汇量测试 → 听力测试 → 口语测试 → 阅读测试 → 写作测试 → 报告生成 → 支付
```

### 3.3 访客 H5 系统

#### 模块职责
- 二维码渠道追踪
- 学校信息展示
- 课程/师资介绍
- 快速预约/咨询

#### 渠道参数设计

```
/h5?channel=poster&school=001&teacher=1001&activity=spring2024
```

| 参数 | 说明 |
|------|------|
| channel | 渠道类型：poster/flyer/namecard/banner/event |
| school | 学校ID |
| teacher | 推广教师ID |
| activity | 活动ID |

## 五、安全架构

### 5.1 身份认证

| 端 | 认证方式 |
|----|----------|
| 小程序端 | 微信授权登录 + 手机号绑定 |
| APP端 | 手机号+验证码 / 账号密码 |
| Web管理端 | JWT Token + RBAC权限 |
| 企业微信端 | 企业微信 OAuth 2.0 |

### 5.2 数据安全

- **传输加密**：全站 HTTPS
- **敏感数据**：AES-256 加密存储
- **儿童信息**：加密存储 + 脱敏展示
- **日志审计**：关键操作全记录

### 5.3 合规措施

- 用户隐私协议
- 儿童个人信息保护（单独授权）
- 摄像头数据家长授权 + 定期清理
- 第三方API合规使用

## 六、部署架构

### 6.1 基础设施

```yaml
环境: Kubernetes (K8S)
  - Web: 3副本
  - API: 5副本
  - AI: 2副本 (GPU)
  - DB: 主从 + 读写分离
  - Cache: Redis Cluster
  - MQ: RabbitMQ Cluster
```

### 6.2 监控告警

| 指标 | 工具 |
|------|------|
| 基础设施监控 | Prometheus + Grafana |
| 应用性能监控 | SkyWalking / Pinpoint |
| 日志收集 | ELK Stack |
| 链路追踪 | Jaeger |
| 告警通知 | 钉钉/企业微信 |

## 七、开发规范

### 7.1 API 规范

- RESTful 风格
- 统一前缀：`/api/v1/`
- 认证方式：JWT Bearer Token
- 错误码：统一错误响应

### 7.2 代码规范

- 前端：ESLint + Prettier
- 后端：PEP 8 + Black
- Git提交：Conventional Commits

### 7.3 数据库规范

- 表名：小写 + 下划线
- 索引：复合索引 + 覆盖索引
- 事务：短事务 + 避免长锁
